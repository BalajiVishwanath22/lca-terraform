PART 4: HOW TO DEPLOY A NEW STACK WITHOUT THE UNWANTED STACKS
================================================================================

Instead of deleting stacks from your existing deployment, the RECOMMENDED
approach is to deploy a fresh LCA CloudFormation stack with parameters that
prevent QNABOT, BEDROCKKB, AGENTASSISTSETUP, and VPCSTACK from being created.

--------------------------------------------------------------------------------
KEY PARAMETER THAT CONTROLS AGENT ASSIST STACKS
--------------------------------------------------------------------------------

The single most important parameter is:

  AgentAssistOption

  Current value:  "QnABot on AWS with new Bedrock knowledge base"
  → This deploys: QNABOT + BEDROCKKB + AGENTASSISTSETUP (all 3 at once)

  To disable all agent assist stacks, set:
  AgentAssistOption = "DISABLED"

  This prevents QNABOT, BEDROCKKB, and AGENTASSISTSETUP from being created.

--------------------------------------------------------------------------------
KEY PARAMETER THAT CONTROLS VPCSTACK
--------------------------------------------------------------------------------

  UseExistingVPC

  Current value: "false"
  → This creates a new VPCSTACK automatically

  Since your audio source (Amazon Connect Contact Lens) does NOT need a VPC,
  the VPC is only created because UseExistingVPC is false and the template
  creates one by default.

  To skip VPC creation, the LCA template uses a condition "ShouldCreateVPC".
  When CallAudioSource = "Amazon Connect Contact Lens", no VPC is technically
  needed. However, the current template always creates VPCSTACK unless you
  provide an existing VPC.

  Option A — Use existing VPC (if you have one):
    UseExistingVPC  = "true"
    VPC             = "vpc-xxxxxxxxxxxxxxxxx"
    PublicSubnet1   = "subnet-xxxxxxxxxxxxxxxxx"
    PublicSubnet2   = "subnet-xxxxxxxxxxxxxxxxx"
    PrivateSubnet1  = "subnet-xxxxxxxxxxxxxxxxx"
    PrivateSubnet2  = "subnet-xxxxxxxxxxxxxxxxx"

  Option B — Leave as is (VPCSTACK deploys but costs ~$65-130/month extra)
